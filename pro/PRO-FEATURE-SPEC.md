## Overview

Pro is a **one-time unlock** via access code. No backend, no accounts, no subscription infrastructure. The code is stored in localStorage after validation. All Pro features are already in the codebase or are lightweight additions.

**Access code format:** `PCPRO-[8-char alphanumeric]`
Example: `PCPRO-A7X3M9K2`

**Validation:** Simple client-side hash check. Store 20 valid codes hardcoded (Pete controls the list). When a user enters a code → hash it → match against the stored hash list → if valid, set `localStorage.getItem('pcpro') = '1'` → UI unlocks.

**Why this is fine for MVP:** We're selling at $29.99, not $299. We want zero friction, zero backend complexity. If codes leak, generate a new batch. Gumroad delivers the code PDF — each purchase gets a unique code from the batch.

## Pro Gate Logic

```js
const isPro = () => localStorage.getItem('pcpro') === '1';
```

Add this helper globally. All Pro-gated UI elements check `isPro()` on render.

## Access Code Management

**Batch 1:** Generate 100 unique codes. Pete stores them.
- Format: `PCPRO-[A-Z0-9]{8}`
- Deliver 1 code per Gumroad purchase (Gumroad supports unique codes per sale via "Unique product keys" feature)
- Gumroad docs: Product → Content → Unique keys → Upload list

**Code delivery:** Customer receives PDF with code after Gumroad purchase. PDF content = see `peptide-calc-pro-access-template.md`.

**Revocation:** Not needed at MVP. If abuse occurs, generate batch 2, announce "codes updated" via X.
